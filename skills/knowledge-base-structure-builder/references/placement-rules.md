# Placement Rules

## Formal Knowledge

Place in `01_Knowledge` only if the content is reviewed, reusable, source-backed or internally evidenced, and has clear scope plus risks/boundaries.

Suitable:

- stable patterns
- reusable mechanisms
- verified workflows
- common failure modes
- stable integration constraints
- decision heuristics

## Project Space

Place in `02_Projects` when content is bound to a project, codebase, experiment, decision, implementation, validation, or review.

Suitable:

- requirements
- design plans
- implementation notes
- experiments
- debug records
- current document groups
- maintenance records

## Inbox

Place in `03_Inbox` when content is unclear, unreviewed, temporary, or potentially useful but not yet promoted.

Suitable:

- candidate summaries
- temporary excerpts
- uncertain notes
- unreviewed external summaries

## Sources

Place in `04_Sources` when preserving source evidence or external material.

Suitable:

- original excerpts
- source evidence cards
- paper/PDF reading notes
- web/source summaries with links

## Archive

Place in `90_Archive` when content is obsolete, frozen, historical, or no longer maintained.

## Runtime Exception

Runtime config, skill/plugin files, and agent capability discovery paths may be read for runtime discovery when authorized, but do not treat them as knowledge context unless explicitly included under normal placement rules.
